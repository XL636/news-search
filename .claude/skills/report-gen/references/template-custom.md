# 自定义关键词报告 — 元指令模板

> 本文件是"元模板"，不定义固定章节结构。当用户使用自定义关键词（如 "AI coding"、"会计"、"Rust"）时，
> Claude 读取本文件，分析关键词属于什么领域，自适应生成报告结构。

## 执行流程

### Step 1: 分析关键词
分析用户输入的关键词，判断它：
- 属于哪个或哪几个预定义领域（AI/ML, DevTools, Security, Cloud, Hardware, Web, Mobile, Data, Blockchain, Biotech, Other）
- 是否跨多个领域（如 "AI coding" 横跨 AI/ML + DevTools）
- 是否是全新领域（如 "会计" 不属于任何预定义领域）

### Step 2: 构建报告结构
根据分析结果，混合或创建章节结构：

**情况 A: 关键词属于单一领域**
→ 使用该领域的模板结构，但将数据筛选为关键词匹配项

**情况 B: 关键词跨多个领域**
→ 混合相关领域的核心章节。例如 "AI coding"：
1. 执行摘要
2. AI 编程工具与助手（来自 AI/ML 模板）
3. 代码生成模型（来自 AI/ML 模板）
4. IDE 集成与开发体验（来自 DevTools 模板）
5. 开源项目亮点（来自 DevTools 模板）
6. 数据概览

**情况 C: 关键词是全新领域**
→ 使用以下通用结构：
1. 执行摘要
2. 核心动态（按热度排序的主要条目）
3. 工具与资源
4. 行业背景
5. 趋势分析
6. 数据概览

### Step 3: 报告标题
- heading1: "{date} {关键词}领域情报日报"
- subtitle: "数据源: GitHub, Hacker News, RSS | 关键词: {keyword} | 共 {total_items} 条"

## 数据过滤规则
自定义关键词通过以下方式匹配数据：
- title LIKE '%keyword%'
- description LIKE '%keyword%'
- tags 包含 keyword
- 多个关键词之间用空格分隔，取 OR 关系

## 写作指引
- 以关键词为核心视角组织内容
- 如果匹配条目少于 3 条，在执行摘要中说明数据有限，建议扩大关键词范围
- 保持与预定义领域模板相同的专业深度
- 关键词翻译：如果关键词是英文，报告标题用英文原名；如果是中文，标题用中文
